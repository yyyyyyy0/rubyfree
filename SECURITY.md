# セキュリティ方針

## 脆弱性の報告

セキュリティ上の問題を見つけた場合は、公開 Issue ではなく GitHub の **Security Advisories**（リポジトリの "Security" タブ → "Report a vulnerability"）から非公開で報告してください。

## このアプリの権限と設計上の前提

rubyfree はアクセシビリティ（他アプリのテキスト読み取り）と、任意で画面収録（OCR）という強力な権限を要求します。これらは攻撃対象領域が広いため、以下の設計で正当化・緩和しています。

- **完全ローカル・非通信**: ネットワーク API を含まず、取得データを外部送信しません。多層で担保します。
  1. ネットワーク API を import・呼び出ししない（CI の grep 入口ガード）
  2. ビルド成果物のシンボル / リンク / エンタイトルメント監査（`Scripts/audit-binary.sh`: `nm -u` / `otool -L` / `codesign -d --entitlements`）
  3. オフライン（機内モード）での全機能動作確認
  4. システム全体のアウトバウンド通信監視
  5. ディスク痕跡・クラッシュレポートへのユーザーデータ非混入確認
- **非永続**: 取得テキスト・画像・読みをディスク / ログに残しません。
- **最小権限**: 既定はアクセシビリティのみ。画面収録（OCR）はオプトイン・Just-In-Time 要求。
- **取得抑止**: セキュアテキストフィールド・除外アプリ上では取得しません。
- **Hardened Runtime**: `com.apple.security.network.client` を付与せず、ライブラリバリデーションを無効化しません（不正 dylib 注入を防止）。

## 配布物の真正性

- 推奨導入はソースからのビルドです。
- バイナリ配布には SHA-256 を併記します。隔離属性の一括解除（`xattr -d`）は案内しません。
